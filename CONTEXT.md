# Never Four

Never Four is a personal publishing context for showing a bounded set of things without turning them into tasks or records.

## Language

**Personal Instance**:
A privately operated installation owned by one person, even when its pages are publicly viewable.
_Avoid_: Hosted product, multi-user workspace, team account

**事不过三**:
The Chinese product name shown as the default public brand.
_Avoid_: Never Four as the default page brand

**Never Four**:
The English product name used for repository, package, and technical references.
_Avoid_: Three Things

**Current Three**:
The set of up to three things the owner is choosing to show right now. It is not a daily archive and does not reset automatically.
_Avoid_: Today's tasks, daily log, history

**Item**:
A single thing shown in a set, such as one activity, paper, book, movie, person, article, or repository. An item has display text and may have a link.
_Avoid_: Task, todo, record

**Set**:
A titled group of up to three items shown together on one public page.
_Avoid_: List, board, channel, collection

**Set Key**:
A short owner-chosen name that identifies a set, such as `papers`, `books`, or `repos`.
_Avoid_: Route, slug, category

**Replace Set**:
The act of publishing a new complete set of zero to three items, replacing whatever was shown before.
_Avoid_: Add item, edit item, delete item, sync item

**Three Limit**:
The rule that a set may contain zero, one, two, or three items, and never more.
_Avoid_: Auto-trimmed list, suggested count

**Empty Slot**:
A visible place in a set where an item could be shown, without implying that the viewer can add one.
_Avoid_: Add button, missing task
